New Hire Onboarding & Equipment Setup SOP Published v1

Process Owner: Angela Tino, Executive Operations Lead

Accountable Executive: Bionka Randolph, CEO/Founder

Last Updated: July 22, 2026 · Review Cadence: Quarterly, or as team tooling changes

Download original Word document

Purpose

This SOP documents the steps taken once a candidate accepts an offer, from document collection through system access and first-week setup, so every new hire has a consistent, complete start experience.

Scope

Begins once Bionka Randolph confirms a candidate has accepted the offer and a start date is set. Ends once the new hire has email, Slack, directory records, and recurring meeting access in place. Does not cover the hiring/interview process itself — see the Hiring & Recruitment SOP for that.

Open gap: Engineering and Product team-specific equipment or software setup (e.g. dev environment access, design tool licenses) is not yet fully documented. Confirm with Jagun Rimes (Engineering) and Dixon Lin (Product) whether team-specific onboarding checklists exist, and incorporate them here.

Process Flow (Summary)

  1. Bionka Randolph or Angela Tino confirms candidate acceptance and requests required documents (ID, etc.) from them
  2. Bionka Randolph informs Angela Tino of the confirmed start date
  3. Angela Tino sets up the new hire's email signature
  4. Angela Tino sends the company directory forms to the new hire
  5. Bionka Randolph sets up and sends Slack access
  6. Angela Tino sends invitations to recurring weekly team meeting(s)
  7. [Gap to confirm] Relevant Team Head sets up any team-specific equipment/tool access

Detailed Steps

Step 1: Document Collection

Who: Bionka Randolph or Angela Tino
When: Immediately after informing the candidate they've been selected
How: Request required documents (e.g. valid ID) from the candidate, needed before their start date.
Output: Completed document set on file.

Step 2: Start Date Confirmation

Who: Bionka Randolph → Angela Tino
How: Bionka informs Angela of the new hire's confirmed start date, which triggers the remaining onboarding steps below.
Output: Onboarding timeline set.

Step 3: Email Signature Setup

Who: Angela Tino
How: Create and configure the new hire's email signature ahead of their start date.
Output: Email signature ready for new hire's first day.

Step 4: Directory Forms

Who: Angela Tino
How: Send the company directory forms to the new hire for completion (contact info, emergency contact, etc.).
Output: New hire added to company directory records.

Step 5: Slack Access

Who: Bionka Randolph
How: Bionka sets up and sends Slack workspace access to the new hire.
Output: New hire has Slack access ahead of/on start date.

Step 6: Recurring Meeting Access

Who: Angela Tino
How: Send calendar invitations for the weekly team meeting(s) relevant to the new hire's team.
Output: New hire is on the recurring meeting calendar.

Step 7: Team-Specific Equipment/Tool Setup (Gap — To Be Confirmed)

Who: Jagun Rimes (Engineering) or Dixon Lin (Product), TBD
How: To be defined — confirm with each Team Head whether they have laptop/hardware provisioning, dev environment access, design tool licenses (e.g. Figma), or other setup steps that should be added here.
Output: Pending — follow up with Engineering and Product leads.

Exceptions and Edge Cases
